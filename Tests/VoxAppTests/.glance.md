## Purpose
This directory contains unit and integration tests for the VoxMac application's core audio processing, user interface state, and system integration components. It validates audio device management, encoding and encryption workflows, recording logic, and accessibility features for the Heads-Up Display (HUD). The suite ensures correct state transitions, file format integrity, and error handling across audio-related operations.

## Key Roles
- **AudioDeviceObserverTests.swift & AudioDeviceManagerTests**: Validate audio input device discovery, selection, and availability monitoring.
- **AudioEncoderTests.swift**: Tests conversion of audio to Opus format, ensuring compression efficiency and file format compliance.
- **AudioFileEncryptionTests.swift**: Verifies secure file handling through encryption/decryption round-trips, key generation, and key zeroization.
- **AudioRecorderTests.swift**: Validates audio level computation, sample rate conversion, and backend selection logic between `AVAudioEngine` and `AVAudioRecorder`.
- **CapturedAudioInspectorTests.swift**: Ensures captured CAF files contain valid audio frames and handles unreadable or header-only files.
- **HUDStateTests.swift & HUDAccessibilityTests.swift**: Validate state machine transitions and VoiceOver accessibility semantics for the application's overlay.
- **HotkeyRegistrationResultTests.swift**: Tests success and failure logic for system-wide hotkey registration and associated error reporting.

## Dependencies and Caveats
- Tests utilize `AVFoundation` for audio processing and both `XCTest` and the Swift `Testing` framework.
- `AudioDeviceObserverTests` are designed to skip execution in environments without available audio input devices.
- `AudioEncoderTests` perform a probe to verify if Opus encoding is supported on the host system before proceeding.
- `AudioRecorder` backend selection can be influenced by the `VOX_AUDIO_BACKEND` environment variable.
- Encryption tests specifically exercise multi-chunk paths for files exceeding 64KB.
- `AudioRecorder` tests address a specific format contract to prevent crashes related to `AVAudioFile` processing format mismatches.
- `CapturedAudioInspector` validation logic is restricted to files with the `.caf` extension.
- `AudioRecorder` includes logic to rebuild converters if input formats become stale during a session.
