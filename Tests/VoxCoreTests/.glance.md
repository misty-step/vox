## Purpose
This directory contains the test suite for the `VoxCore` framework, focusing on the validation of Speech-to-Text (STT) provider orchestration, text rewriting logic, and core utility components. The tests verify reliability strategies such as retries, fallbacks, hedging, and concurrency limits, as well as quality assurance mechanisms for processed text.

## Key Roles
- **STT Orchestration Tests**: Files like `RetryingSTTProviderTests`, `FallbackSTTProviderTests`, `HedgedSTTProviderTests`, and `HealthAwareSTTProviderTests` validate complex execution strategies for STT services, including error-based retries, racing multiple providers, and health-based routing.
- **Rewrite Logic Tests**: `ModelRoutedRewriteProviderTests` and `FallbackRewriteProviderTests` verify the routing of text transformation requests based on model identifiers (e.g., Gemini, Mercury) and provider availability.
- **Quality Assurance**: `RewriteQualityGateTests` validates the logic used to accept or reject rewritten text based on character ratios, Levenshtein similarity, and content word overlap.
- **Core Models and Errors**: `ProcessingLevelTests`, `STTErrorTests`, and `VoxErrorTests` ensure the integrity of framework-wide enums and error handling behavior.
- **Infrastructure and Utilities**: `MultipartFormDataTests`, `SecureFileDeleterTests`, and `ConcurrencyLimitedSTTProviderTests` test low-level data formatting, filesystem cleanup, and asynchronous resource management.

## Dependencies and Caveats
- **Test Frameworks**: Utilizes both `XCTest` and the Swift `Testing` library.
- **STT Reliability**: `RetryingSTTProvider` implements exponential backoff and distinguishes between retryable (e.g., throttled) and non-retryable (e.g., invalid audio) errors.
- **Dynamic Timeouts**: `TimeoutSTTProvider` calculates deadlines dynamically based on audio file size using a base timeout and a "seconds per MB" multiplier.
- **Legacy Support**: `ProcessingLevel` includes logic to migrate legacy string values (e.g., "off", "light", "aggressive") to current enum cases.
- **Health Monitoring**: `HealthAwareSTTProvider` tracks provider performance using a rolling window of samples to calculate success rates and average latency.
- **Task Management**: `HedgedSTTProvider` and `ConcurrencyLimitedSTTProvider` rely on Swift Concurrency (Tasks and Continuations) to manage permit acquisition and task cancellation.
- **Quality Metrics**: The `RewriteQualityGate` applies different minimum character ratios depending on the `ProcessingLevel` (e.g., 0.6 for `.clean`, 0.3 for `.polish`).
- **File Integrity**: `SecureFileDeleter` is designed to handle missing or already-deleted files without raising errors.
