## Purpose
This directory contains unit tests for the `VoxUI` module, focusing on the validation of UI-driven state management, user preferences, and visual rendering logic. The tests ensure the correctness of system menu bar components, onboarding flows, hotkey availability, and the debug workbench used for tracking request lifecycles.

## Key Roles
- **DebugWorkbenchStoreTests.swift & DebugWorkbenchSinkTests.swift**: Validate the capturing, storage, and thread-safe forwarding of request data, including logs, transcripts, and processing levels.
- **StatusBarMenuSnapshotTests.swift & StatusBarIconRendererTests.swift**: Verify the presentation logic for the macOS status bar, including menu item titles, recovery actions, and pixel-aligned icon geometry.
- **PreferencesStoreCacheTests.swift & PreferencesStoreDefaultLevelTests.swift**: Ensure API key status caching correctly prioritizes environment variables over Keychain and determines default processing levels based on configuration.
- **HotkeyStateTests.swift**: Tests the mutation and publication of hotkey availability states and the execution of retry closures.
- **OnboardingStoreTests.swift & OnboardingSessionExtensionTests.swift**: Confirm the persistence of onboarding flags in `UserDefaults` and the tracking of completion events.
- **SettingsViewContentTests.swift**: Validates UI text formatting for version information and service provider summaries within the settings interface.
- **ShareVoxTests.swift**: Validates the integrity of repository URLs and clipboard strings for sharing features.

## Dependencies and Caveats
- Depends on the `VoxCore`, `VoxUI`, `VoxMac`, and `VoxDiagnostics` modules.
- Utilizes the Swift `Testing` framework and `Combine` for asynchronous state observation.
- Relies on `UserDefaults` for persisting state across suite executions, using unique suite names for isolation.
- Interacts with `NSPasteboard` to verify clipboard operations.
- Environment variables such as `VOX_DEBUG_WORKBENCH` and various API key strings (e.g., `OPENROUTER_API_KEY`) explicitly influence runtime logic and cache states.
- `DebugWorkbenchStore` implements a hard-coded log tail limit of 150 entries.
- Status bar rendering is constrained by specific constants defined in `BrandIdentity`, such as `menuIconSize` and `menuIconStrokeWidth`.
- Logic for selecting `.clean` versus `.raw` processing levels is dependent on the presence of configured rewrite keys.
