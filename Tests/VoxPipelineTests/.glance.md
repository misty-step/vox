## Purpose
This directory contains the test suite for the `VoxPipeline` module, focusing on the orchestration of speech-to-text (STT) transcription, text rewriting, and final insertion. It validates the end-to-end dictation flow, including audio validation, format conversion, and error handling. The suite also covers system prompt generation, result caching, and performance benchmarking.

## Key Roles
- `DictationPipelineTests.swift`: Validates the core pipeline logic, including STT integration, rewriting, pasting, and fallback mechanisms.
- `DictationPipelineConcurrencyTests.swift`: Ensures thread safety and verifies that preference access occurs on the `MainActor`.
- `DictationPipelineDiagnosticsTests.swift`: Tests the diagnostic logging system for tracking stage outcomes, cache hits, and conversion fallbacks.
- `PipelineBenchmarkTests.swift`: Measures execution latency and framework overhead against defined P50 and P95 latency budgets.
- `RewriteResultCacheTests.swift`: Validates the TTL-based caching mechanism, capacity-based eviction, and key isolation by model and processing level.
- `LastDictationRecoveryStoreTests.swift`: Tests the persistence, retrieval, and expiration of dictation snapshots for recovery.
- `RewritePromptsTests.swift`: Verifies the content and formatting instructions of system prompts for different processing levels.

## Dependencies and Caveats
- Depends on `VoxCore` for domain models and `VoxPipeline` for the implementation under test.
- Utilizes `AVFoundation` for generating and validating test audio files (CAF format).
- Benchmarks require the `VOX_RUN_BENCHMARK_TESTS` environment variable to be set to "1" for execution.
- The pipeline includes an optional Opus conversion stage that triggers based on file size thresholds.
- Rewrite stages utilize configurable timeouts; the pipeline falls back to raw transcripts if these limits are exceeded.
- Cache entries and recovery snapshots are subject to TTL (Time To Live) expiration.
- Uses `@testable` imports to access internal components like `RewriteResultCache` and `LastDictationRecoveryStore`.
- Mock providers are used extensively to simulate STT and rewriting latencies and failures.
