## Purpose
This directory contains unit and integration tests for the `VoxSession` component, focusing on its lifecycle, dependency injection, and interaction with audio recording, processing pipelines, and UI feedback. It verifies the session's ability to coordinate batch and streaming Speech-to-Text (STT) workflows, including error handling and recovery mechanisms. The tests also validate the session's integration with encrypted audio storage and external session extensions.

## Key Roles
- **VoxSessionTests.swift**: Implements a comprehensive test suite using the `Testing` framework. It defines various mock objects—including `MockRecorder`, `MockPipeline`, `MockHUD`, and `MockStreamingProvider`—to simulate audio hardware, processing backends, and user interface components for isolated verification of `VoxSession` logic.

## Dependencies and Caveats
- Depends on `VoxCore`, `VoxPipeline`, `VoxMac`, and `VoxSession` modules.
- Utilizes the Swift `Testing` framework for test execution and assertions.
- Tests environmental configuration parsing for features such as `VOX_DISABLE_STREAMING_STT`, `VOX_STT_ROUTING`, and `VOX_AUDIO_BACKEND`.
- Verifies the decryption of audio files before transcription when using `EncryptedAudioRecording` and `AudioFileEncryption`.
- Simulates asynchronous streaming STT scenarios, including handling for timeouts, chunk buffering, and residual data draining.
- Validates integration with `SessionExtension` for recording authorization, completion telemetry, and failure reporting.
- Includes tests for recovery features, such as copying raw transcripts to the clipboard and retrying rewrites using `LastDictationRecoveryStore`.
- Mocks provide granular control over asynchronous behaviors, such as simulating non-cooperative thread blocking during streaming finalization.
