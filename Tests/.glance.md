## Purpose
This directory contains the comprehensive test suite for the Vox application and its supporting frameworks, including `VoxCore`, `VoxMac`, `VoxDiagnostics`, `VoxPerfAuditKit`, `VoxPipeline`, and `VoxUI`. It validates audio processing workflows, speech-to-text (STT) orchestration strategies, text rewriting logic, and user interface state transitions. The suite ensures the integrity of system integrations, such as audio device management, hotkey registration, and encrypted file handling. Additionally, it covers diagnostic logging, performance auditing, and service provider communication across various network and hardware scenarios.

## Key Roles
- **Audio and Recording Tests**: Validate device discovery, Opus encoding, encryption/decryption round-trips, and backend selection between `AVAudioEngine` and `AVAudioRecorder`.
- **STT and Rewrite Orchestration**: Verify reliability strategies like retries, fallbacks, hedging, and quality gates for processed text.
- **Pipeline and Session Management**: Test the end-to-end dictation flow, including concurrency safety, result caching, and session lifecycle coordination.
- **UI and Accessibility**: Ensure correct state machine transitions for the HUD, status bar rendering, and VoiceOver semantics.
- **Diagnostics and Performance**: Validate error code mapping, log rotation, ZIP export integrity, and statistical distribution for performance audits.
- **Provider Clients**: Test REST and WebSocket communication for services like Deepgram and Gemini using `URLProtocol` stubs.

## Dependencies and Caveats
- Utilizes both `XCTest` and the Swift `Testing` framework for test execution and assertions.
- Depends on system utilities like `/usr/bin/afconvert` for audio conversion and `/usr/bin/ditto` for ZIP validation.
- Requires specific environment variables, such as `DEEPGRAM_API_KEY` and `VOX_AUDIO_BACKEND`, for integration and provider-specific tests.
- `AudioDeviceObserverTests` and `AudioEncoderTests` include logic to skip execution or probe for support based on host hardware availability.
- `VoxPipeline` benchmarks are gated by the `VOX_RUN_BENCHMARK_TESTS` environment variable.
- Audio processing tests address specific format contracts to prevent `AVAudioFile` mismatches and restrict validation to `.caf` files.
- `RewriteQualityGate` logic applies variable character ratio thresholds based on the selected `ProcessingLevel`.
- Caching and recovery components, such as `RewriteResultCache`, implement TTL-based expiration and capacity-based eviction.
