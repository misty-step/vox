## Purpose
This directory contains the unit test suite for the `VoxPerfAuditKit` framework. It validates the logic for command-line argument parsing, environment variable processing, and statistical data distribution. The tests ensure that performance audit configurations and provider plans are correctly resolved and that error handling is triggered for invalid inputs or missing requirements.

## Key Roles
- **VoxPerfAuditKitTests.swift**: Houses multiple test suites, including `PerfAuditConfigTests` for input validation, `StageDistributionTests` for mathematical accuracy of percentiles, and `PerfProviderPlanTests` for service provider selection logic.

## Dependencies and Caveats
- **Testing Framework**: Utilizes the Swift `Testing` library for defining suites, tests, and expectations.
- **Environment Variable Dependencies**: Tests rely on specific environment variables such as `GITHUB_SHA` for metadata and various API keys (`OPENROUTER_API_KEY`, `ELEVENLABS_API_KEY`, `DEEPGRAM_API_KEY`, `GEMINI_API_KEY`) for provider resolution.
- **Required Arguments**: `PerfAuditConfig` requires `--audio` and `--output` arguments to initialize successfully.
- **Error Handling**: The suite verifies specific `PerfAuditError` cases, including `helpRequested`, `invalidArgument` (for lanes or negative warmups), and `missingRequiredKey`.
- **Provider Selection**: `PerfProviderPlan` resolution logic supports both "auto" selection and "forced" provider overrides via the `VOX_PERF_STT_PROVIDER` variable.
- **Statistical Logic**: `StageDistribution` is tested for its ability to handle empty samples, single samples, and p50/p95 interpolation for multiple samples.
- **Case Insensitivity**: The provider resolution logic is verified to handle case-insensitive string matching for forced provider IDs.
