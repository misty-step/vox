## Purpose
This directory provides the configuration framework, provider orchestration logic, and statistical utilities for a performance auditing system. It facilitates the parsing of execution parameters, determines service provider strategies for Speech-to-Text (STT) and text rewriting based on environment state, and calculates distribution metrics for performance data.

## Key Roles
- **PerfAuditConfig.swift**: Defines the configuration structure, handles command-line argument parsing, and manages error reporting for the audit process.
- **PerfProviderPlan.swift**: Resolves the operational plan for STT and rewrite services by evaluating available API keys and environment-defined provider preferences.
- **StageDistribution.swift**: Provides a data structure and logic for calculating statistical distributions, including p50, p95, minimum, and maximum values from numeric samples.

## Dependencies and Caveats
- Depends on the `Foundation` framework for file URL resolution, JSON coding, and basic data types.
- Requires `ELEVENLABS_API_KEY` or `DEEPGRAM_API_KEY` to be present in the environment to initialize the STT chain.
- Mandatory requirement for `OPENROUTER_API_KEY` within the provider resolution logic.
- Rewrite routing is conditionally set to "model-routed" if `GEMINI_API_KEY` or `INCEPTION_API_KEY` is detected, otherwise defaulting to "openrouter".
- STT provider selection is restricted to the identifiers `elevenlabs` and `deepgram`.
- Audit lanes are explicitly limited to `provider` and `codepath` types.
- Statistical calculations in `StageDistribution` return zeroed metrics if the provided sample set is empty.
- Configuration initialization automatically attempts to source the `commitSHA` from the `GITHUB_SHA` environment variable.
