## Purpose
The `VoxSession` directory manages the lifecycle and orchestration of voice dictation sessions. It coordinates audio recording, Speech-to-Text (STT) transcription, and text rewriting while transitioning through idle, recording, and processing states. The directory provides logic for both real-time streaming and batch processing, incorporating error handling, diagnostic logging, and session recovery.

## Key Roles
- `VoxSession`: The primary controller that manages recording states, initializes dictation pipelines, and coordinates between audio hardware, STT providers, and the user interface.
- `StreamingAudioBridge`: An internal coordinator that buffers and enqueues audio chunks from the recorder to an active streaming session, handling asynchronous setup and finalization.
- `StreamingSessionPump`: An actor-based helper that manages the lifecycle of a streaming STT session, including sending audio data and collecting partial transcripts.
- `DictationPipeline`: (Constructed within `VoxSession`) Orchestrates the sequential processing of audio through STT, rewriting, and clipboard pasting.

## Dependencies and Caveats
- Depends on internal modules including `VoxCore`, `VoxDiagnostics`, `VoxMac`, `VoxPipeline`, `VoxProviders`, and `VoxUI`.
- Requires microphone access permissions; dictation fails if permissions are denied.
- Supports streaming STT through ElevenLabs or Deepgram, with configurable setup and finalization timeouts.
- Uses a fallback mechanism for STT, prioritizing cloud providers before falling back to local `AppleSpeechClient` or `SpeechTranscriberClient` (on macOS 26.0+).
- Implements optional "hedged routing" for STT, which initiates parallel requests with staggered delays.
- Logic is influenced by specific environment variables, including `VOX_STT_ROUTING`, `VOX_DISABLE_STREAMING_STT`, `VOX_MAX_CONCURRENT_STT`, and `VOX_AUDIO_BACKEND`.
- Handles encrypted audio capture, requiring successful decryption via `AudioFileEncryption` before batch processing.
- Includes a recovery system via `LastDictationRecoveryStore` to retry rewrites or retrieve the most recent raw transcript.
