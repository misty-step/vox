## Purpose
This directory provides a framework for capturing, storing, and exporting application diagnostics and performance telemetry. It manages structured event logging to local JSONL files with rotation policies and supports asynchronous remote ingestion of performance data. The module also aggregates system context, application metadata, and error mappings for troubleshooting and analysis.

## Key Roles
- **DiagnosticsStore**: An actor that manages local persistent logging of `DiagnosticsEvent` objects. It handles file rotation, pruning of old logs, and exports logs and system context into a ZIP archive.
- **PerformanceIngestClient**: A telemetry client that asynchronously buffers and uploads allowlisted performance events to a remote endpoint via NDJSON.
- **DiagnosticsContext**: Aggregates environment and application state, including OS version, STT configurations, and the presence of API keys for external services.
- **ProductInfo**: Resolves application versioning, build numbers, and attribution details from the main bundle or environment variables.
- **DiagnosticsValue**: A codable enum that provides a unified representation for string, integer, double, and boolean data within diagnostic fields.

## Dependencies and Caveats
- Depends on `Foundation` and `VoxCore`.
- Uses the `/usr/bin/ditto` system utility to compress diagnostic exports.
- `PerformanceIngestClient` requires the `VOX_PERF_INGEST_URL` environment variable to be defined for remote telemetry.
- Performance ingestion is "best-effort," dropping buffered events on upload failure to avoid blocking execution paths.
- Local log rotation is constrained by configurable byte limits and a maximum number of rotated files.
- Remote telemetry is restricted to an allowlist of event names, such as `pipeline_timing` and `share_triggered`.
- Error reporting utilizes manual mapping of internal types (e.g., `STTError`, `AudioConversionError`) to string-based diagnostic codes.
- Environment variables like `VOX_STT_ROUTING`, `VOX_AUDIO_BACKEND`, and `VOX_MAX_CONCURRENT_STT` influence the captured `DiagnosticsContext`.
