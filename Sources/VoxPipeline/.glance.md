## Purpose
The `VoxPipeline` directory orchestrates the end-to-end dictation workflow, transforming audio or text inputs into finalized, pasted text. It manages a multi-stage process that includes optional audio encoding, speech-to-text (STT) transcription, AI-driven rewriting, and performance tracking. The directory also provides infrastructure for result caching and short-term transcript recovery.

## Key Roles
- **`DictationPipeline`**: The primary coordinator that sequences STT, rewriting, and pasting operations while managing audio validation and Opus conversion.
- **`LastDictationRecoveryStore`**: An actor that maintains a time-limited snapshot of the most recent dictation, including raw and final text, for recovery purposes.
- **`RewritePrompts`**: A utility that generates specialized system instructions for "clean" and "polish" processing levels to ensure transcripts are edited without following embedded commands.
- **`RewriteResultCache`**: An actor providing in-memory storage for previous rewrite results to avoid redundant LLM requests for identical inputs.
- **`PipelineTiming`**: A structure used to capture, calculate, and format latency metrics for each stage of the processing lifecycle.

## Dependencies and Caveats
- Depends on `VoxCore` for essential protocols (`STTProvider`, `RewriteProvider`, `TextPaster`) and data types (`ProcessingLevel`).
- Implements a task-group-based timeout mechanism to cap the duration of pipeline and rewrite stages.
- Performs conditional CAF-to-Opus audio conversion based on file size thresholds and provider availability.
- The rewrite stage defaults to the raw transcript if the LLM provider times out, fails, or returns empty results.
- Both `LastDictationRecoveryStore` and `RewriteResultCache` utilize a default 600-second time-to-live (TTL) for data pruning.
- Caching is restricted to transcripts and results that fall under a 1,024-character limit.
- Employs explicit prompt engineering to prevent the rewriting engine from interpreting transcript content as actionable instructions.
- Uses `CFAbsoluteTime` for high-precision measurement of encode, STT, rewrite, and paste intervals.
