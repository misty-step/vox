## Purpose
This directory provides macOS-specific infrastructure for audio capture, processing, and user interface management. It includes components for managing CoreAudio devices, recording audio via `AVAudioEngine` or `AVAudioRecorder`, and encoding outputs to Opus. Additionally, it handles system-level integrations such as global hotkeys, keychain-based credential storage, and clipboard manipulation for automated text insertion.

## Key Roles
- **AudioRecorder.swift**: Orchestrates audio capture using either `AVAudioEngine` or `AVAudioRecorder`, performing format conversion to 16kHz mono PCM and providing real-time metering.
- **AudioEncoder.swift**: Interfaces with the system `afconvert` utility to encode audio files into Opus-in-CAF format at a 24 kbps bitrate.
- **AudioFileEncryption.swift**: Implements AES-GCM encryption and decryption for audio files using `CryptoKit` with chunk-based processing and key zeroization.
- **HUDController.swift & HUDView.swift**: Manage and render a floating SwiftUI-based head-up display that tracks recording duration, audio levels, and processing states.
- **HotkeyMonitor.swift**: Handles registration and callbacks for system-wide keyboard shortcuts using Carbon APIs.
- **ClipboardPaster.swift**: Facilitates text insertion by manipulating the system pasteboard and simulating Cmd+V keystrokes via AppleScript or `CGEvent`.
- **AudioDeviceManager.swift**: Monitors CoreAudio hardware changes and manages input device selection and validation.
- **KeychainHelper.swift**: Provides a wrapper for the macOS Keychain to securely store and retrieve API keys for integrated services.

## Dependencies and Caveats
- Dependencies include `AVFoundation`, `CoreAudio`, `AudioToolbox`, `CryptoKit`, `Carbon`, `AppKit`, `SwiftUI`, and `VoxCore`.
- Requires the system binary `/usr/bin/afconvert` for Opus encoding operations.
- Automated pasting functionality requires the application to be trusted for Accessibility in system settings.
- Audio recording backends can be toggled via the `VOX_AUDIO_BACKEND` environment variable.
- Supports an opt-in per-app audio routing mode via the `VOX_ENABLE_PER_APP_AUDIO_ROUTING` environment variable.
- Encrypted files use a specific "VOXENC1" magic header and a 256-bit symmetric key.
- HUD position and visibility are persisted using `UserDefaults`.
- `CapturedAudioInspector` validates CAF files to ensure they contain decodable frames rather than just container headers.
