## Purpose
The `Sources` directory contains the modular codebase for a macOS dictation and text-processing application. It integrates system-level audio capture, encryption, and clipboard management with various local and cloud-based AI providers for speech-to-text and text rewriting. The directory structure supports the full application lifecycle, encompassing entry point initialization, user interface management, performance benchmarking, and diagnostic telemetry.

## Key Roles
- **VoxApp & VoxAppKit**: Manage the application entry point, lifecycle orchestration, and core system-level service coordination.
- **VoxSession & VoxPipeline**: Orchestrate the end-to-end dictation workflow, managing state transitions and the sequencing of transcription and rewriting stages.
- **VoxCore & VoxProviders**: Define foundational protocols and provide specific client implementations for local and cloud-based AI services.
- **VoxMac & VoxUI**: Handle macOS-specific hardware integration (audio, hotkeys, keychain) and the SwiftUI-based user interface components.
- **VoxDiagnostics, VoxBenchmark, & VoxPerfAudit**: Provide infrastructure for telemetry, performance auditing, and LLM quality evaluation.

## Dependencies and Caveats
- Relies on `AppKit`, `AVFoundation`, `CoreAudio`, and `Carbon` for macOS system integration and hardware control.
- Requires system-level Accessibility and Microphone permissions for core functionality like audio capture and automated text insertion.
- Integrates with external APIs including OpenRouter, Deepgram, ElevenLabs, Gemini, and InceptionLabs.
- Uses the system `/usr/bin/afconvert` utility for Opus audio encoding and `CryptoKit` for AES-GCM file encryption.
- Implements a fallback mechanism for STT, transitioning from cloud providers to local Apple speech services.
- Utilizes the macOS Keychain for secure storage of API credentials and `UserDefaults` for preference persistence.
- Performance telemetry and debugging tools are conditionally activated via environment variables or `DEBUG` build flags.
- Supports both real-time streaming and batch processing modes for speech-to-text transcription.
