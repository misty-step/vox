## Purpose
The `Sources/VoxAppKit` directory manages the core application lifecycle and system-level integrations for the macOS environment. It coordinates the initialization of session logic, diagnostic reporting, and user interface components like the status bar and settings windows. The directory serves as the entry point for configuring hardware permissions, global hotkeys, and service provider credentials.

## Key Roles
- **AppDelegate.swift**: Orchestrates application startup, including the initialization of `VoxSession`, `StatusBarController`, and window controllers for onboarding and settings. It manages global hotkey registration, triggers diagnostic logging, handles system permission prompts, and performs automated cleanup of legacy recovery audio files.

## Dependencies and Caveats
- Depends on internal modules `VoxCore`, `VoxDiagnostics`, `VoxMac`, `VoxSession`, and `VoxUI` for core logic and interface elements.
- Utilizes `AppKit` for application framework support and `Carbon` for low-level hotkey management.
- Requires system-level Accessibility and Microphone permissions to function correctly.
- Automatically deletes recovery audio files older than 24 hours from the application support directory upon launch.
- Attempts to register Option+Space as a global hotkey; failure to register triggers a warning and provides manual retry options.
- Validates API key presence for ElevenLabs, Deepgram, Gemini, OpenRouter, and Inception, applying model fallback logic if specific keys are missing.
- Includes a conditional debug workbench and sink that are only activated in `DEBUG` builds when the workbench runtime is enabled.
- Monitors `VoxSession` state changes to update the system status bar across idle, recording, and processing modes.
