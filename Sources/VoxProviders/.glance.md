## Purpose
This directory provides a suite of Speech-to-Text (STT) and text-rewriting provider implementations. It includes clients for local Apple speech services and cloud-based APIs such as Deepgram, ElevenLabs, Gemini, and OpenRouter. The directory also contains utilities for audio format conversion and a central assembly point for configuring provider chains, instrumentation, and retry logic.

## Key Roles
- **AppleSpeechClient & SpeechTranscriberClient**: Implement Apple-native speech recognition using `SFSpeechRecognizer` or the macOS 26+ `SpeechTranscriber` pipeline.
- **DeepgramClient & ElevenLabsClient**: Provide batch and streaming STT interfaces via cloud APIs and WebSocket transports.
- **GeminiClient, InceptionLabsClient, & OpenRouterClient**: Implement the `RewriteProvider` interface for text processing, with OpenRouter supporting multi-model fallback chains.
- **AudioConverter**: Manages audio file conversion to Opus or WAV formats by wrapping the system `/usr/bin/afconvert` utility.
- **ProviderAssembly**: Functions as a central factory for constructing instrumented and retrying provider chains based on provided API keys.
- **URLSession+PhaseAwareSTTTimeout**: Extends `URLSession` to provide specialized upload tracking that distinguishes between network stalls and server-side processing delays.

## Dependencies and Caveats
- `AppleSpeechClient` requires `NSSpeechRecognitionUsageDescription` to be defined in the application bundle to avoid crashes during authorization.
- `SpeechTranscriberClient` is gated to macOS 26.0+ and requires the `FoundationModels` framework.
- `AudioConverter` depends on the existence and executability of `/usr/bin/afconvert` and probes its help output to verify Opus support.
- Streaming providers utilize `URLSessionWebSocketTask` for real-time communication and require non-empty API keys for session initialization.
- `OpenRouterClient` implements a fallback mechanism that can retry requests with relaxed provider parameters if strict routing fails.
- Batch STT clients calculate processing timeouts dynamically based on the byte size of the uploaded audio file.
- `ProviderAssembly` enforces specific retry limits: three attempts for ElevenLabs and two for Deepgram.
- Audio conversion processes in `AudioConverter` are subject to a hardcoded 30-second timeout.
