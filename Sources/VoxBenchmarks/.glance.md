## Purpose
This directory contains a benchmarking utility for evaluating LLM performance on text rewriting tasks. It automates the process of sending transcripts from a JSON corpus to multiple models via the OpenRouter API to measure quality, latency, and cost. The tool generates statistical summaries and model recommendations based on predefined quality targets for specific processing levels.

## Key Roles
- `main.swift`: The primary executable entry point that handles command-line configuration, orchestrates the benchmark execution loop, manages API communication with OpenRouter, and generates both JSON and Markdown reports.

## Dependencies and Caveats
- Depends on `VoxCore`, `VoxPipeline`, and `VoxProviders` for processing levels, quality evaluation logic, and error types.
- Requires a valid `OPENROUTER_API_KEY` environment variable for API authentication.
- Relies on the OpenRouter API for model inference and usage reporting.
- Requires a JSON corpus file (version 1) containing transcript entries and associated processing levels.
- Supports specific command-line options for corpus paths, output locations, iteration counts, and model selection.
- Benchmark logic is restricted to the `.clean` and `.polish` processing levels.
- Implements a retry mechanism with exponential backoff for network, timeout, and throttling errors.
- Quality assessment is performed using the `RewriteQualityGate` utility to determine pass/fail rates and quality ratios.
