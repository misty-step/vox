## Purpose
This directory manages the user interface and state for the application's initial setup and configuration process. It provides a checklist for users to verify system permissions, configure cloud-based transcription or rewriting services, and complete a test dictation. The components ensure the application has the necessary accessibility and microphone access required for core functionality.

## Key Roles
- **OnboardingChecklistView.swift**: A SwiftUI view that displays the status of required permissions (Accessibility, Microphone) and optional configurations (Cloud keys, Microphone selection).
- **OnboardingSessionExtension.swift**: A `SessionExtension` that updates the onboarding state upon the successful completion of a dictation event.
- **OnboardingStore.swift**: An observable object that manages and persists onboarding progress, such as whether the checklist has been shown or the first dictation completed, using `UserDefaults`.
- **OnboardingWindowController.swift**: An `NSWindowController` that hosts the onboarding view within a standard macOS window.

## Dependencies and Caveats
- Depends on `VoxMac` for `PermissionManager` and `SystemSettingsLink` to handle macOS privacy permissions.
- Depends on `VoxCore` for `SessionExtension` and `DictationUsageEvent` protocols and types.
- Requires Accessibility permissions to perform text insertion into other applications.
- Checks for specific cloud provider keys in `PreferencesStore`, including ElevenLabs, Deepgram, Gemini, OpenRouter, and Inception.
- Permission statuses are refreshed automatically when the application becomes active via `NSApplication.didBecomeActiveNotification`.
- Functionality is categorized into "Raw" (always available via Apple Speech), "Clean," and "Polish" modes, with the latter two requiring cloud API keys.
- Onboarding state is persisted using the `UserDefaults` keys `onboarding.hasShownChecklist` and `onboarding.hasCompletedFirstDictation`.
