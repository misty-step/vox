## Purpose
This directory provides the user interface and state management for application settings. It facilitates the configuration of global hotkeys, audio input devices, and cloud provider API keys for transcription and text rewriting services.

## Key Roles
- **PreferencesStore.swift**: Acts as the central state manager for user settings, persisting data to `UserDefaults` and `KeychainHelper` while supporting environment variable overrides.
- **SettingsView.swift**: Serves as the primary SwiftUI container, coordinating the layout of settings sections and the cloud keys management sheet.
- **CloudProviderCatalog.swift**: Defines the registry of supported cloud services—such as ElevenLabs, Deepgram, Gemini, Inception, and OpenRouter—and generates configuration summaries.
- **BasicsSection.swift**: Manages hardware-related configurations, including microphone selection via `AudioDeviceObserver` and hotkey status display.
- **CloudKeysSheet.swift**: Provides a secure interface for entering API keys, using a local buffer to ensure keys are only committed to the store upon user confirmation or sheet dismissal.
- **HotkeyState.swift**: Tracks the registration availability of the global application shortcut and provides a mechanism for retry attempts.
- **SettingsSection.swift**: Implements a reusable UI container for grouping related settings with support for primary and secondary prominence levels.

## Dependencies and Caveats
- Depends on `VoxMac` for audio device observation and keychain access.
- Relies on `VoxCore` for processing level definitions and `VoxDiagnostics` for product metadata and versioning.
- API keys are stored in the macOS Keychain; in debug builds, environment variables override these stored values.
- Transcription defaults to Apple Speech (on-device) if no cloud provider keys are configured.
- The default processing level is determined by the presence of rewrite-capable API keys (e.g., Gemini, OpenRouter, or Inception).
- Hotkey status is monitored; if "Option+Space" is unavailable, the UI provides a retry button and suggests alternative start methods.
- The `PreferencesStore` maintains a `keyStatusCache` to prevent synchronous Keychain reads during SwiftUI view re-renders.
- Microphone selection includes a fallback to the system default if the specifically selected device is unavailable.
