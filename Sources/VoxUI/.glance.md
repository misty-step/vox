## Purpose
This directory provides the comprehensive user interface layer for the application on macOS. It manages the system status bar integration, application settings, onboarding workflows for system permissions, and a diagnostic workbench for monitoring processing requests. The components coordinate state between user interactions and the underlying core logic for dictation and text rewriting.

## Key Roles
- **StatusBarController**: Manages the macOS system status item, updating the menu and icon based on recording state, processing levels, and service readiness.
- **StatusBarIconRenderer**: Renders dynamic status bar icons using `NSBezierPath`, varying the monogram style (open V, outlined, or filled triangle) based on application state.
- **SettingsWindowController**: An `NSWindowController` that hosts the settings interface and manages the state of global hotkey availability.
- **DebugWorkbench (Subdirectory)**: Provides a diagnostic interface for real-time monitoring of request statuses, execution logs, and transcript data.
- **Onboarding (Subdirectory)**: Handles the initial setup flow, including macOS accessibility and microphone permission verification.
- **Settings (Subdirectory)**: Manages hardware selection, hotkey registration, and secure API key storage for cloud transcription and rewriting services.
- **ShareVox**: Defines the repository URL and plain-text strings used for sharing the application via the clipboard.

## Dependencies and Caveats
- Depends on `VoxCore` for `ProcessingLevel` definitions, session extensions, and dictation event types.
- Depends on `VoxMac` for `PermissionManager`, `AudioDeviceObserver`, and `KeychainHelper`.
- Utilizes `AppKit` for window management, menu bar interaction, and `NSBezierPath` graphics.
- Relies on `VoxDiagnostics` for product metadata and exporting diagnostic ZIP files via `NSSavePanel`.
- API keys for cloud providers (e.g., Deepgram, Gemini, OpenRouter) are stored in the macOS Keychain, with `PreferencesStore` caching statuses to avoid synchronous reads during UI renders.
- The Debug Workbench is only enabled in `#if DEBUG` builds when specific environment variables (e.g., `VOX_DEBUG_WORKBENCH`) are set.
- Status bar icons adjust their visual stroke width based on the active `ProcessingLevel`.
- Onboarding state and user preferences are persisted using `UserDefaults`.
