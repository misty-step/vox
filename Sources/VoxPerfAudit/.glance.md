## Purpose
This directory contains the entry point and core logic for a performance auditing tool designed to benchmark the dictation pipeline. It executes automated test cycles across multiple processing levels and lanes to collect timing distributions and provider usage metrics, ultimately generating a structured JSON report.

## Key Roles
- **VoxPerfAudit (main)**: Orchestrates the audit lifecycle, including environment configuration, audio fixture loading, pipeline execution for warmup and measurement phases, and JSON result encoding.
- **PerfPreferences**: Implements preference reading by merging system environment variables with local `.env` files to provide API keys for various AI providers.
- **ProviderUsageRecorder**: Provides thread-safe tracking of STT and rewrite provider invocations to snapshot model usage during the audit.
- **TimingCollector**: Aggregates pipeline timing data (encode, STT, rewrite, and paste durations) using thread-safe locking.
- **Instrumented Providers**: Wraps standard STT and Rewrite providers to intercept calls for usage recording without altering underlying logic.
- **Fixed Delay Providers**: Mock implementations used in the `codepath` lane to provide deterministic, simulated latency for benchmarking pipeline overhead.

## Dependencies and Caveats
- **Internal Frameworks**: Depends on `VoxPerfAuditKit`, `VoxAppKit`, `VoxCore`, `VoxPipeline`, and `VoxProviders`.
- **Environment Requirements**: Requires valid API keys for ElevenLabs, OpenRouter, Deepgram, Gemini, and Inception to run the `provider` lane.
- **Concurrency**: Uses `NSLock` to protect timing and usage data across asynchronous task boundaries.
- **Input Handling**: Processes audio fixtures from provided URLs, calculating file size and generating unique fixture identifiers.
- **Report Schema**: Generates a versioned JSON output (schema version 3) containing statistical distributions for stage-specific latencies.
- **Mocking**: Employs a `NoopPaster` to suppress actual text insertion during performance tests.
- **STT Configuration**: Supports forced provider reordering and concurrency limits via the `VOX_MAX_CONCURRENT_STT` environment variable.
