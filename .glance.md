## Purpose
This directory serves as the root for Vox, a modular macOS application designed for dictation and AI-driven text processing. The system captures system-level audio, transcribes it through a resilient chain of cloud and local providers, and optionally rewrites the result into refined prose. It utilizes a "Bring Your Own Key" (BYOK) model and emphasizes a zero-configuration, simplicity-first user experience.

## Key Roles
- **Package.swift**: Defines the modular target architecture, including core protocols (`VoxCore`), provider clients (`VoxProviders`), macOS integrations (`VoxMac`), and the application entry point.
- **README.md & project.md**: Document the project vision, functional features like global hotkeys and auto-pasting, and high-level architectural patterns such as decorator composition.
- **CLAUDE.md & AGENTS.md**: Provide technical guidance on the module dependency DAG, security boundaries, and the "Definition of Done" for development.
- **CONTRIBUTING.md & CHANGELOG.md**: Outline the contribution lifecycle, automated release procedures using Landfall, and a detailed history of versioned features and fixes.
- **Info.plist**: Declares mandatory system-level usage descriptions for microphone access and speech recognition fallback.

## Dependencies and Caveats
- Requires macOS 14+ and Swift 5.9; specific features like `SpeechTranscriberClient` require macOS 26+ SDK availability.
- Mandates system Accessibility permissions for automated text insertion and Microphone access for audio capture.
- Implements a resilient STT chain using sequential fallback (ElevenLabs to Deepgram to Apple Speech) or optional hedged routing.
- Utilizes a strict module hierarchy where `VoxCore` contains shared protocols and errors with no external dependencies.
- Requires explicit `AVAudioFile` initialization parameters to prevent system crashes on specific macOS versions when handling Int16 interleaved buffers.
- Enforces a "simplicity gate" ADR that restricts the addition of new user-facing settings or advanced configuration UI.
- Secures API credentials using the macOS Keychain, with support for `.env.local` environment variable overrides during development.
- Automates versioning and releases through the Landfall pipeline, requiring conventional commit messages for changelog generation.
