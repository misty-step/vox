## Purpose
This directory provides a suite of tools for performance auditing, quality assurance, and automated release management for the Vox application. It includes scripts for executing benchmarks, verifying code coverage, and linting both source code and CI workflows. Additionally, the directory contains utilities for evaluating external Speech-to-Text and LLM providers to track latency and quality metrics. These tools support continuous integration by generating standardized reports and managing GitHub Pull Request communications.

## Key Roles
- `benchmark.sh`: Executes pipeline latency benchmarks through specific Swift test filters.
- `coverage-report.sh`: Generates Markdown coverage summaries and enforces per-module percentage thresholds.
- `lint.sh` and `lint-cerberus-workflow.sh`: Perform Swift source linting and validate GitHub Action workflow configurations and secret naming policies.
- `post-coverage-comment.sh`: Manages persistent coverage report comments on GitHub Pull Requests using HTML markers for deduplication.
- `release-macos.sh`: Automates the build, signing, and notarization process for the macOS application bundle.
- `rewrite-bakeoff.py` and `stt-eval.py`: Compare performance, cost, and quality metrics across various LLM and Speech-to-Text providers.
- `run-tests-ci.sh`: Manages Swift test execution within CI environments, including timeout handling and process tree cleanup.
- `Info.plist.template`: Provides the metadata structure and system permission declarations for the macOS application.

## Dependencies and Caveats
- Requires the GitHub CLI (`gh`) and `jq` for API interactions, artifact retrieval, and JSON processing.
- macOS-specific build utilities including `codesign`, `notarytool`, `stapler`, and `spctl` are required for application releases.
- Python 3 is required for data processing, coverage analysis, and provider evaluations.
- External API interactions depend on environment variables such as `GH_TOKEN`, `OPENROUTER_API_KEY`, and specific provider keys for Deepgram, OpenAI, and Groq.
- Coverage reporting enforces specific thresholds for core modules while excluding hardware-dependent or UI-heavy modules like `VoxMac` and `VoxUI`.
- Release metadata is managed through placeholders in `Info.plist.template` (e.g., `__VOX_BUNDLE_ID__`) that require substitution during the build process.
- CI test execution utilizes `CI_TEST_TIMEOUT_SECONDS` for lifecycle management and supports conditional integration tests via `VOX_ENABLE_INTEGRATION_TESTS`.
- Workflow linting enforces a dedicated secret naming policy, specifically requiring `CERBERUS_OPENROUTER_API_KEY`.
