## Purpose
This directory provides comprehensive technical documentation, architectural decisions, and design specifications for the Vox macOS application. It serves as a central repository for Architecture Decision Records (ADRs), performance benchmarks, and UI design explorations. The documentation details the system's modular Swift architecture, speech-to-text (STT) resilience strategies, and release procedures. Additionally, it contains postmortem reports and technical specifications for dictation pipeline optimizations.

## Key Roles
- **ARCHITECTURE.md**: Defines the layered Swift package structure, STT resilience chain decorators, and the core data flow from audio capture to text insertion.
- **CODEBASE_MAP.md**: Provides a high-level system overview, module guide, and mapping of environment variables to system behaviors.
- **MODEL_EVALUATION.md**: Compares Large Language Models (LLMs) from Gemini and OpenRouter based on latency, cost, and quality metrics for text rewriting.
- **RELEASE.md**: Outlines the Landfall automated versioning process and manual macOS signing and notarization requirements.
- **TRIAD_PLAYBOOK.md**: Establishes a decision-making framework to balance output quality, user simplicity, and processing speed.
- **adr/**: Documents significant design choices, such as the "simplicity-first" UI gate and the use of the `SessionExtension` protocol for lifecycle hooks.
- **performance/**: Stores latency budgets, test corpora, and historical bakeoff results used to select optimal STT and rewrite providers.
- **postmortems/**: Records technical root causes and architectural hardening measures, such as payload validation guards, following production-impacting incidents.

## Dependencies and Caveats
- The system utilizes `AVAudioEngine` as the default capture backend, with an opt-out path to `AVAudioRecorder` managed via environment variables.
- STT routing follows a sequential fallback chain (ElevenLabs to Deepgram to Apple Speech) to minimize API costs and complexity.
- Rewriting is model-routed, prioritizing direct Gemini API access with OpenRouter as a fallback path for non-Google models.
- Production rewrites do not use automated quality gates for rejection; quality regression detection is shifted to evaluation suites and manual observation.
- The application requires macOS 14+, Swift 5.9, and specific TCC permissions for microphone, speech recognition, and accessibility.
- Release procedures require an Apple Developer Program membership and configured notarytool credentials for Gatekeeper compliance.
- Privacy is maintained by ensuring diagnostics logs exclude transcript text and API keys, recording only counts, booleans, and timings.
- The "Vox HUD" UI is constrained to a fixed 170x40px size and utilizes a hybrid breathing dot with asymmetric EMA smoothing for audio level feedback.
